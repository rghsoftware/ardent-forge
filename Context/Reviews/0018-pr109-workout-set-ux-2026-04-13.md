#### [FIX] P18-001: Swipe-to-delete has no DB persistence

- **Detail:** `onDeleteSet` calls the Zustand `deleteSet` store action directly. The store removes the set from in-memory state only; the adapter is never called. The user sees the set disappear, but on page reload or crash recovery it reappears from the database. `useDeleteLoggedSet` exists in `use-workout-logs.ts` but is not called from this path. Fix: add a coordinated `deleteSet` to `use-active-workout.ts` that calls the mutation before updating the store, and wire `onDeleteSet` through it.
- **Status:** âś… Fixed
- **Resolution:** Added coordinated `deleteSet` to `use-active-workout.ts` that awaits `useDeleteLoggedSet` mutation before calling the store action. `log.$workoutId.tsx` `onDeleteSet` now calls the bridge function and surfaces failures via `setPageError`.

#### [FIX] P18-005: Triple error logging on `unconfirmSet` violates Hook Log Ownership rule

- **Resolution:** `useUpdateLoggedSet`'s `onError` (already in place) remains the sole log owner. Removed the duplicate `console.error` from the `unconfirmSet` catch in `use-active-workout.ts` (re-throws with an explanatory comment). Changed `handleUnconfirmSet` in the page to `catch {}` with only `setPageError` (no re-log).

#### [FIX] P18-006: Silent guard clauses in user-action handlers
- **Files:** `src/routes/_authenticated/log.$workoutId.tsx:293` (handleFinish), `src/routes/_authenticated/log.$workoutId.tsx:378` (handleConfirmSet)
- **Severity:** High
