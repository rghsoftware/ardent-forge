## Issue #32 -- event_metadata column comment formatting

**Goal:** Rewrite the `session_templates.event_metadata` column comment in a follow-up migration to eliminate the mixed quoted/unquoted pseudo-schema syntax.

**Current (ambiguous):**

```sql
'JSONB event metadata for EVENT templates. Expected shape: '
'{"location": string, "eventDate": ISO-8601, "distance": {"value": number, "unit": "mi"|"km"}, '
'"cutoffTime": {"seconds": number}, "elevation": {"value": number, "unit": "ft"|"m"}}. '
'NULL for non-EVENT categories.';
```

**Approach:**

1. Create `supabase/migrations/<next_timestamp>_fix_event_metadata_comments.sql`
2. Use `$$` dollar quoting and a plaintext field list:

```sql
comment on column session_templates.event_metadata is
$$JSONB event metadata for EVENT session templates. NULL for non-EVENT categories.

Fields:
  location    text    -- event venue or location name
  eventDate   text    -- ISO-8601 date string (e.g. "2026-06-15")
  distance    jsonb   -- { value: number, unit: "mi" | "km" }
  cutoffTime  jsonb   -- { seconds: number }
  elevation   jsonb   -- { value: number, unit: "ft" | "m" }
$$;
```

3. Close issue #32.

**Verification:** `bunx supabase db push` applies cleanly locally.

**Risks:** Comment-only migration -- no schema changes, zero risk.
